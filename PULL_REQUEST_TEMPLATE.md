## Template Usage Guidelines

### For Bug Fixes
- Clearly describe the bug being fixed
- Include steps to reproduce the original issue
- Explain how the fix resolves the problem
- Include tests that verify the fix

### For New Features
- Describe the feature and its purpose
- Explain how it fits into the existing codebase
- Include comprehensive tests
- Update relevant documentation

### For Refactoring
- Explain the motivation for refactoring
- Describe what was changed and why
- Ensure all tests still pass
- Maintain backward compatibility

### For Documentation Updates
- Clearly describe what documentation was updated
- Explain why the changes were necessary
- Ensure accuracy and completeness
